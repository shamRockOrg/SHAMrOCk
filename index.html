Sphero HAnd MOtion Control

The aim of the SHAMrOCk project is to create a simple, general-purpose control system, initially used to control a Sphero spherical robot using the LeapMotion hands-free controller system.

Initial problems

The connection with Sphero isn't stable neither on linux nor on windows. For Linux the bluetooth stack and the python biding might not be complete but on windows it should work.

The main problem is that we cannot get anything from the ball, we can send a command but we get no response, ever.

We will check the java project from the last group who used the Sphero to see if it still working and if the java driver work better than the python/ruby one.

Kåre orders a Sphero V2 which, according to the documentation, is better than V1 when it cme to sdk and connection.

Starting our project!

First of all, we calc an average position of our hand when we were using LeapMotion so the control would be smoother. Every 5 frames we get the average value of our hands position.

Then, we focused on the "non-official" Sphero driver in order to receive data; We had to modified to non blocking. Because it used to block when we receive the first message.

Thank Python! We imported Sphero libraries to the LeapMotion controller, we wrote some commands for connecting, and 'voila', we can controll Sphero from LeapMotion. Now, we have to improve the control to make it more intuitive and smoother.

Current status of the project

Pervasive Computing has finished and we are happy what we have achieved.

We can control sphero with our hand through LeapMotion. The ball has 360º of movement, how do we do that? First of all, we calculate a start point of our hand, then we are able to measure a direction and speed from our hand next position every frame getting the angle and length of the vector. Finally, we send the right command.

The color of sphero will change depending on the speed.

As we have gesture recognition in LeapMotion, when we roll our finger and sphero is stopped we modify the header so it points us.

Finally, we can control both versions although V2 is smoother.

Further work

If we continue the project, we would improve and develop the next points:

  • Auto detect sphero version.
  • Detect collisions.
  • New ways to control sphero like hand rotation instead of movement.
  • Similar controls for different stuff like a drone.
